# UI Contract for Agents

This file summarizes the UI contract for agents.

The canonical visual references are:

- `DESIGN.md`
- `风格.md`

## Required UI Direction

The frontend must combine:

- Apple-style structure and discipline
- liquid glass as the elevated interaction layer

## UI Rules

- use Apple-like hierarchy, spacing, and restraint
- use glass selectively on navigation, overlays, floating controls, and key elevated cards
- keep content surfaces readable and controlled
- prefer premium clarity over trendy effects

## UI Anti-Patterns

Do not:

- overuse glass
- use random visual styles page by page
- build generic AI-dashboard aesthetics
- replace hierarchy with decoration
- use loud gradients or multiple competing accent colors

## Before Shipping UI Changes

Check:

1. Is the structure clear and Apple-like?
2. Is glass only used on elevated interactive layers?
3. Is the page still readable without effects?
4. Is the palette restrained?
5. Does the motion remain subtle?
