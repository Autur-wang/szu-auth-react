# SZU Sports Frontend Style Guide

This file defines the project-specific UI style for the SZU Sports frontend.  
It is separate from `DESIGN.md`.

`DESIGN.md` is the broad Apple-inspired reference.  
This file is the practical design contract for this repository.

## 1. Design Direction

The frontend should combine two ideas:

- Apple-style structure and discipline
- Liquid glass as a foreground material

This means the interface should feel calm, premium, minimal, and precise.  
It should not feel flashy, noisy, playful, or overloaded.

The visual goal is:

- clean information hierarchy
- large areas of controlled whitespace
- restrained typography
- dark/light neutral surfaces
- glass only where elevation or interaction matters

## 2. Core Principle

Use Apple for the layout language.  
Use liquid glass for the interactive layer.

In practice:

- page structure should be Apple-like
- navigation, floating controls, overlays, and special cards may use glass
- content sections should stay readable and solid
- not every surface should be transparent or blurred

If everything is glass, the page becomes cheap and unclear.

## 3. Visual Tone

The product should feel:

- premium
- focused
- modern
- athletic but not aggressive
- technical but not cold

The page should look like a serious product interface for sports reservation, not a gaming dashboard or a marketing landing page full of effects.

## 4. Layout Rules

- Prefer strong section-based layout.
- Use large spacing between major sections.
- Keep content centered and controlled.
- Avoid cluttered multi-column density unless the content really requires it.
- Use clear visual rhythm between hero, status, controls, and detail areas.

Recommended structure:

1. top glass navigation or header
2. hero or summary section
3. main functional card area
4. secondary information panels
5. logs, history, or operational detail below

## 5. Typography Rules

- Typography must feel crisp and intentional.
- Use Apple-like hierarchy: large display heading, restrained section titles, clean body text.
- Headlines should be short and strong.
- Body copy should be compact and readable.
- Avoid decorative or playful fonts.

Preferred type behavior:

- large bold display titles for hero content
- medium-weight titles for sections
- regular-weight text for body and labels
- small, muted labels for supporting metadata

Avoid:

- oversized paragraph blocks
- too many font sizes
- exaggerated letter spacing
- trendy “startup” typography styles

## 6. Color Rules

Base the interface on a restrained Apple-like palette:

- black or near-black
- white
- soft gray
- one clear action blue

Suggested color behavior:

- dark hero or deep backdrop for premium atmosphere
- light or soft-gray content areas for readability
- blue only for actions, links, focus states, and key confirmations

Avoid:

- rainbow gradients everywhere
- purple-heavy AI-style palettes
- overly saturated neon accents
- too many competing highlight colors

## 7. Liquid Glass Rules

Liquid glass is allowed, but only in the right places.

Good places for glass:

- top navigation
- floating command bar
- login or control card
- modal dialogs
- segmented controls
- action trays
- status overlays

Bad places for glass:

- long-form content backgrounds
- large text reading areas
- every card on the page
- all buttons at once
- entire page background

Glass should create depth, not chaos.

## 8. Background Rules

The background should support the foreground, not compete with it.

Use:

- deep dark photo overlays
- soft neutral gradients
- subtle field textures
- low-contrast environmental visuals

Do not use:

- busy illustrations behind text
- loud mesh gradients
- strong color noise
- animated backgrounds that fight with content

For this project, sports imagery is acceptable, but it must be darkened and controlled so the UI stays readable.

## 9. Component Rules

### Navigation

- Use a compact glass header or pill-shaped bar.
- Keep actions minimal.
- Make navigation feel light and floating.

### Cards

- Primary cards may use liquid glass.
- Secondary cards should often be solid or semi-solid.
- Card content should have clear spacing and clean alignment.

### Buttons

- Buttons should be simple, rounded, and confident.
- Primary actions should use the project blue.
- Secondary actions should be subtle, not noisy.
- Do not add excessive gradients to buttons.

### Inputs

- Inputs should feel precise and calm.
- Use soft borders, soft contrast, and clean spacing.
- Keep focus states clear and accessible.

### Dialogs

- Dialogs are a strong use case for glass.
- Use blur, depth, and edge highlights carefully.
- Keep dialog content minimal and structured.

## 10. Motion Rules

Motion should feel soft and meaningful.

Use:

- subtle fade-in
- gentle upward reveal
- small hover lift
- restrained glass shimmer or refraction
- smooth transitions between states

Avoid:

- bouncing UI
- exaggerated spring effects everywhere
- constant looping motion
- flashy micro-animations on every element

Motion should support hierarchy, not draw attention to itself.

## 11. Content Style

- Keep copy short.
- Prefer direct product language.
- Avoid overly promotional wording.
- Avoid fake futuristic language.
- Prefer clarity over hype.

Good tone:

- direct
- calm
- premium
- useful

Bad tone:

- exaggerated
- “AI product launch” style hype
- too casual
- too cute

## 12. What the UI Should Feel Like

The final interface should feel like:

- Apple product page discipline
- premium control panel clarity
- glass used as an interaction layer
- a real sports reservation product, not a concept toy

## 13. Hard Constraints

When designing new frontend pages in this repository:

- do not use random visual styles page by page
- do not mix multiple unrelated design languages
- do not overuse glass
- do not replace hierarchy with decoration
- do not make the page look like a generic AI dashboard
- do not make the page feel game-like unless the feature explicitly needs it

## 14. Implementation Standard

Before building a new page or component, check:

1. Is the structure Apple-like and clear?
2. Is glass used only on the elevated interactive layer?
3. Is the page readable without relying on effects?
4. Is the color palette restrained?
5. Does the motion remain subtle?
6. Does the result feel premium instead of trendy?

If the answer to any of these is no, redesign before implementation.
