F-04  自动抢场执行流程

系统最核心的差异化功能。基于 Bull Queue + Redis 的异步任务调度，每分钟扫描活跃规则并执行抢场逻辑，包含重试、并发控制与异常告警。
flowchart TD
    A([Cron 每分钟触发]) --> B[查询 auto_booking_rules\n条件: is_active=true\n且 target_date 匹配]
    B --> C{有待执行规则？}
    C -->|否| Z([本轮结束])
    C -->|是| D[逐条入队 Bull Queue]
    D --> E[Worker 取出任务]
    E --> F{用户会员有效？}
    F -->|否| G[跳过 + 停用规则\n发送会员到期通知]
    F -->|是| H[解密用户学号凭证]
    H --> I[调用深大体育API\n模拟登录]
    I --> J{登录成功？}
    J -->|失败| K[记录失败日志\n告警运营团队]
    J -->|是| L[查询目标场地\n目标时段列表]
    L --> M{目标时段存在且可预约？}
    M -->|否| N{距离开放还有多久？}
    N -->|超过30分钟| O[标记等待\n1小时后重试]
    N -->|30分钟内| P[高频模式: 30秒轮询]
    P --> L
    M -->|是| Q[发送预约请求]
    Q --> R{预约成功？}
    R -->|是| S[写入bookings表]
    S --> T[自动扣款/冻结]
    T --> U[推送成功通知\n微信+站内]
    U --> V[更新 success_count+1]
    V --> W([任务完成])
    R -->|失败| X{已重试次数}
    X -->|小于3次| Q
    X -->|达到上限| Y[标记规则失败\n发送失败通知]

系统时序图 · Sequence Diagrams
描述关键场景下各系统组件之间的交互时序，包括自动抢场调度链路、支付回调链路与学号认证链路。
UML Sequence · 最核心场景

S-01  自动抢场完整时序

展示从定时调度触发到用户收到推送通知的完整调用链。涉及 Cron → Bull Queue → Worker → 深大体育API → MySQL → 通知服务 六个组件。
sequenceDiagram
    autonumber
    participant CR as ⏰ Cron调度
    participant Q as 📦 Bull Queue(Redis)
    participant W as ⚙️ Worker进程
    participant DB as 🗄️ MySQL数据库
    participant SZU as 🏫 深大体育系统
    participant NS as 📣 通知服务

    CR->>DB: 查询 active 规则列表
    DB-->>CR: 返回规则数组
    loop 每条规则
        CR->>Q: enqueue(ruleId, priority)
    end
    Q->>W: 分发任务
    W->>DB: 获取规则详情 + 用户凭证
    DB-->>W: 规则 + 加密凭证
    W->>W: AES解密学号密码
    W->>SZU: POST /api/login {studentId, pwd}
    SZU-->>W: sessionToken
    W->>SZU: GET /api/venues/{id}/slots?date=X
    SZU-->>W: 时段列表
    alt 目标时段可预约
        W->>SZU: POST /api/bookings {slotId, token}
        SZU-->>W: 预约成功 {bookingNo}
        W->>DB: INSERT bookings (status=paid)
        W->>DB: UPDATE time_slots (status=booked)
        W->>DB: UPDATE auto_booking_rules (success_count+1)
        W->>NS: triggerNotification(userId, SUCCESS)
        NS-->>用户: 微信公众号推送 + 站内通知
    else 目标时段已满
        W->>Q: delay(300s) 重新入队
        Q-->>W: 5分钟后重试
    else 登录失败
        W->>DB: INSERT 错误日志
        W->>NS: triggerAlert(ADMIN, LOGIN_FAIL)
    end
UML Sequence · 支付链路

S-02  会员购买支付时序

展示用户发起会员购买到支付完成的完整支付链路，包括支付网关异步回调的 Webhook 处理与幂等性校验。
sequenceDiagram
    autonumber
    participant U as 👤 用户浏览器
    participant FE as 🖥️ 前端
    participant API as 🔧 后端API
    participant DB as 🗄️ MySQL
    participant PAY as 💳 支付网关
    participant WS as 🔌 WebSocket

    U->>FE: 点击「立即购买」
    FE->>API: POST /membership/orders {plan: monthly}
    API->>DB: INSERT membership_orders (status=pending)
    DB-->>API: orderId
    API->>PAY: 创建支付单 {orderId, amount, notifyUrl}
    PAY-->>API: {payUrl, qrcode}
    API-->>FE: 返回支付二维码
    FE->>U: 展示支付弹窗+二维码
    U->>PAY: 扫码完成支付
    PAY->>API: POST /webhooks/payment (异步回调)
    API->>API: 验签 + 幂等校验
    API->>DB: UPDATE membership_orders (status=paid)
    API->>DB: UPDATE users (membership_plan, expires_at)
    API->>DB: INSERT notifications (type=membership)
    API->>WS: broadcast(userId, PAYMENT_SUCCESS)
    WS-->>FE: 实时推送成功消息
    FE->>U: 显示「会员开通成功」弹窗
    API-->>PAY: 200 OK (确认回调已处理)
UML Sequence · 账号体系

S-03  深大学号验证时序

用户注册第二步的学号绑定流程。后端通过模拟登录深大体育系统验证真实性，成功后加密存储凭证用于后续自动抢场。
sequenceDiagram
    autonumber
    participant U as 👤 用户
    participant API as 🔧 后端API
    participant CACHE as ⚡ Redis缓存
    participant SZU as 🏫 深大体育系统
    participant DB as 🗄️ MySQL

    U->>API: POST /students/verify {studentId, szuPassword}
    API->>CACHE: 检查频率限制 (用户IP + userId)
    CACHE-->>API: 剩余次数
    alt 触达上限(3次/15min)
        API-->>U: 429 Too Many Requests
    else 正常
        API->>SZU: 模拟POST登录请求
        SZU-->>API: 响应结果
        alt 验证成功
            API->>API: AES-256-GCM 加密 szuPassword
            API->>DB: INSERT student_bindings
            API->>DB: UPDATE users SET is_student_verified=true
            API->>DB: INSERT points_logs (source=verify, delta=+50)
            API-->>U: 200 {verified: true, points: +50}
        else 学号或密码错误
            API->>CACHE: 失败次数+1
            API-->>U: 401 {message: 学号或密码错误}
        else 深大系统无响应
            API-->>U: 503 {message: 校方系统暂时不可用，请稍后重试}
        end
    end

-- ┌─────────────────────────────────────────────────────┐
-- │  Table: users  ·  用户主表                           │
-- │  每个注册用户一条记录，包含认证、会员、积分状态        │
-- └─────────────────────────────────────────────────────┘
CREATE TABLE users (
  id                    BIGINT              NOT NULL AUTO_INCREMENT,
  phone                 VARCHAR(20)         NOT NULL              -- 手机号，登录唯一标识,
  password_hash         VARCHAR(255)        NOT NULL              -- bcrypt hash, cost=12,
  nickname              VARCHAR(50)         DEFAULT NULL,
  avatar_url            VARCHAR(500)        DEFAULT NULL,
  is_student_verified   TINYINT(1)          NOT NULL DEFAULT 0    -- 学号已绑定并验证,
  points                INT                 NOT NULL DEFAULT 0    -- 当前积分余额,
  role                  ENUM('user','admin') NOT NULL DEFAULT 'user',
  membership_plan       ENUM('free','monthly','quarterly','yearly')
                                             NOT NULL DEFAULT 'free',
  membership_expires_at TIMESTAMP           DEFAULT NULL          -- NULL=无会员,
  trial_used            TINYINT(1)          NOT NULL DEFAULT 0    -- 免费体验是否已用,
  referral_code         VARCHAR(16)         DEFAULT NULL          -- 个人邀请码,
  referred_by           BIGINT              DEFAULT NULL          -- 邀请人user_id,
  login_at              TIMESTAMP           DEFAULT NULL          -- 最近登录时间,
  created_at            TIMESTAMP           NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at            TIMESTAMP           NOT NULL DEFAULT CURRENT_TIMESTAMP
                                             ON UPDATE CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  UNIQUE KEY uk_phone (phone),
  UNIQUE KEY uk_referral_code (referral_code),
  KEY idx_membership (membership_plan, membership_expires_at),
  KEY idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='用户主表';
-- ┌─────────────────────────────────────────────────────┐
-- │  Table: student_bindings  ·  深大学号绑定表           │
-- │  存储加密的深大体育系统登录凭证，用于自动抢场          │
-- └─────────────────────────────────────────────────────┘
CREATE TABLE student_bindings (
  id           BIGINT       NOT NULL AUTO_INCREMENT,
  user_id      BIGINT       NOT NULL              -- 关联 users.id，1对1,
  student_id   VARCHAR(20)  NOT NULL              -- 深大学号,
  real_name    VARCHAR(50)  DEFAULT NULL          -- 从深大系统获取的真实姓名,
  college      VARCHAR(100) DEFAULT NULL          -- 学院,
  major        VARCHAR(100) DEFAULT NULL,
  grade        VARCHAR(10)  DEFAULT NULL          -- 年级,
  encrypted_pwd TEXT        NOT NULL              -- AES-256-GCM 加密的深大密码,
  iv           VARCHAR(64)  NOT NULL              -- AES 初始向量,
  verified_at  TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  UNIQUE KEY uk_user_id (user_id),
  UNIQUE KEY uk_student_id (student_id),
  CONSTRAINT fk_sb_user FOREIGN KEY (user_id)
    REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='深大学号绑定与加密凭证';
-- ┌─────────────────────────────────────────────────────┐
-- │  Table: venues  ·  场地表                            │
-- └─────────────────────────────────────────────────────┘
CREATE TABLE venues (
  id             INT          NOT NULL AUTO_INCREMENT,
  name           VARCHAR(100) NOT NULL,
  type           ENUM('basketball','badminton','tennis',
                      'football','swimming','gym','other')
                              NOT NULL,
  location       VARCHAR(200) DEFAULT NULL           -- 位置描述,
  description    TEXT         DEFAULT NULL,
  images         JSON         DEFAULT NULL           -- ["url1","url2",...],
  price_per_hour DECIMAL(8,2) NOT NULL,
  capacity       INT          NOT NULL DEFAULT 1,
  open_time      TIME         DEFAULT '08:00:00',
  close_time     TIME         DEFAULT '22:00:00',
  is_active      TINYINT(1)   NOT NULL DEFAULT 1,
  sort_order     INT          NOT NULL DEFAULT 0     -- 展示排序,
  created_at     TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  KEY idx_type_active (type, is_active)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='运动场地表';
-- ┌─────────────────────────────────────────────────────┐
-- │  Table: time_slots  ·  时间段表                       │
-- │  每天由定时任务生成未来N天的时间段记录                  │
-- └─────────────────────────────────────────────────────┘
CREATE TABLE time_slots (
  id           BIGINT     NOT NULL AUTO_INCREMENT,
  venue_id     INT        NOT NULL,
  slot_date    DATE       NOT NULL,
  start_time   TIME       NOT NULL,
  end_time     TIME       NOT NULL,
  status       ENUM('available','locked','booked','closed')
                           NOT NULL DEFAULT 'available',
  booked_by    BIGINT     DEFAULT NULL         -- 预约用户ID,
  price        DECIMAL(8,2) NOT NULL            -- 允许浮动定价,
  locked_at    TIMESTAMP  DEFAULT NULL         -- 加锁时间（10分钟超时自动释放）,
  locked_by    BIGINT     DEFAULT NULL         -- 锁定用户ID,

  PRIMARY KEY (id),
  UNIQUE KEY uk_venue_date_start (venue_id, slot_date, start_time),
  KEY idx_venue_date (venue_id, slot_date),
  KEY idx_status (status),
  CONSTRAINT fk_ts_venue FOREIGN KEY (venue_id)
    REFERENCES venues(id),
  CONSTRAINT fk_ts_user FOREIGN KEY (booked_by)
    REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='场地时间段（每日生成）';
-- ┌─────────────────────────────────────────────────────┐
-- │  Table: bookings  ·  预约记录主表                     │
-- │  手动预约与自动预约统一记录，含支付状态                │
-- └─────────────────────────────────────────────────────┘
CREATE TABLE bookings (
  id             BIGINT      NOT NULL AUTO_INCREMENT,
  user_id        BIGINT      NOT NULL,
  slot_id        BIGINT      NOT NULL,
  rule_id        BIGINT      DEFAULT NULL           -- 由哪条自动规则触发，手动为NULL,
  type           ENUM('manual','auto')
                             NOT NULL DEFAULT 'manual',
  status         ENUM('pending','paid','cancelled','refunded','completed')
                             NOT NULL DEFAULT 'pending',
  amount         DECIMAL(8,2) NOT NULL,
  discount       DECIMAL(8,2) NOT NULL DEFAULT 0.00   -- 积分抵扣金额,
  payment_method ENUM('wechat','alipay','balance')
                             DEFAULT NULL,
  payment_no     VARCHAR(64) DEFAULT NULL            -- 支付流水号,
  cancel_reason  VARCHAR(200) DEFAULT NULL,
  booked_at      TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP,
  paid_at        TIMESTAMP   DEFAULT NULL,
  cancelled_at   TIMESTAMP   DEFAULT NULL,

  PRIMARY KEY (id),
  UNIQUE KEY uk_payment_no (payment_no),
  KEY idx_user_status (user_id, status),
  KEY idx_slot (slot_id),
  KEY idx_booked_at (booked_at),
  CONSTRAINT fk_bk_user FOREIGN KEY (user_id)
    REFERENCES users(id),
  CONSTRAINT fk_bk_slot FOREIGN KEY (slot_id)
    REFERENCES time_slots(id),
  CONSTRAINT fk_bk_rule FOREIGN KEY (rule_id)
    REFERENCES auto_booking_rules(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='预约记录主表';
-- ┌─────────────────────────────────────────────────────┐
-- │  Table: auto_booking_rules  ·  自动预约规则表         │
-- │  会员专属，Bull Queue 每轮调度从此表读取活跃规则       │
-- └─────────────────────────────────────────────────────┘
CREATE TABLE auto_booking_rules (
  id             BIGINT      NOT NULL AUTO_INCREMENT,
  user_id        BIGINT      NOT NULL,
  venue_id       INT         NOT NULL,
  target_days    VARCHAR(20) NOT NULL DEFAULT '1,2,3,4,5'
                                              -- 周一到周五，逗号分隔的 ISO weekday,
  target_start   TIME        NOT NULL              -- 目标时段开始时间,
  target_end     TIME        NOT NULL,
  advance_days   INT         NOT NULL DEFAULT 3     -- 提前几天开始尝试,
  priority       INT         NOT NULL DEFAULT 1     -- Bull Job 优先级（越小越优先）,
  max_price      DECIMAL(8,2) DEFAULT NULL          -- 最高可接受单价（NULL=不限）,
  is_active      TINYINT(1)  NOT NULL DEFAULT 1,
  success_count  INT         NOT NULL DEFAULT 0,
  fail_count     INT         NOT NULL DEFAULT 0,
  last_run_at    TIMESTAMP   DEFAULT NULL,
  last_success_at TIMESTAMP  DEFAULT NULL,
  created_at     TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  KEY idx_user_active (user_id, is_active),
  KEY idx_venue (venue_id),
  CONSTRAINT fk_abr_user FOREIGN KEY (user_id)
    REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_abr_venue FOREIGN KEY (venue_id)
    REFERENCES venues(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='自动预约规则（会员专属）';

-- ┌─────────────────────────────────────────────────────┐
-- │  Table: points_logs  ·  积分流水表                   │
-- │  users.points 的所有变更都在此记录（不可删改）        │
-- └─────────────────────────────────────────────────────┘
CREATE TABLE points_logs (
  id          BIGINT       NOT NULL AUTO_INCREMENT,
  user_id     BIGINT       NOT NULL,
  delta       INT          NOT NULL              -- 正值=获得, 负值=消耗,
  balance     INT          NOT NULL              -- 变更后的余额快照,
  source      ENUM('register','booking','review','invite',
                   'exchange','admin_adjust','expire')
                            NOT NULL,
  ref_id      VARCHAR(50)  DEFAULT NULL          -- 关联业务主键ID,
  note        VARCHAR(200) DEFAULT NULL          -- 备注说明,
  created_at  TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (id),
  KEY idx_user (user_id),
  KEY idx_source (source),
  CONSTRAINT fk_pl_user FOREIGN KEY (user_id)
    REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='积分变更流水（只增不改）';
